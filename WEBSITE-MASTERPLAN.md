## ОБЩАЯ КОНЦЕПЦИЯ

**Цель сайта:** Генерация лидов → продажа услуги редизайна сайтов  
**Целевое действие:** Заполнение формы (сайт + телефон) для получения бесплатного демо-аудита  
**Аудитория:** Владельцы МСБ с устаревшими сайтами (Барнаул, Новосибирск, Алтай, вся Россия)  
**Тон:** Экспертный, но человечный. Уверенный, но не агрессивный. С эмпатией.

### Ключевой принцип каждого блока

Каждый блок отвечает на один вопрос в голове посетителя:
1. **Hero** → "Чем мне это поможет?"
2. **Проблемы клиента** → "Они понимают мою ситуацию"
3. **Скрытые проблемы** → "Оказывается, всё ещё хуже, чем я думал"
4. **Как решаем** → "У них есть чёткий план"
5. **Тарифы** → "Сколько это стоит и что я получу"
6. **CTA (аудит)** → "Рискнуть нечем, попробую"
7. **Кейсы** → "У них реально получается"
8. **Эмпатия** → "Они такие же люди, им можно доверять"
9. **Footer** → "Они настоящие, всё легально"

### Экспертиза: UX + Конверсия

**Принцип:** Минимум пунктов, максимум конверсии. Навигация — это не каталог, а воронка.

**Структура меню:**
| Элемент | Тип | Назначение |
|---------|-----|-----------|
| Логотип ПАПИНА | Ссылка на верх | Бренд-идентификация |
| Проблемы | Якорь #problems | Вовлечение через боль |
| Как работаем | Якорь #solution | Доверие через процесс |

**Детали реализации:**
- Sticky-навигация (прилипает при скролле)
- На мобильных: бургер-меню + CTA-кнопка "Аудит" всегда видна

- Без пункта "О нас" — вся страница и есть "о нас"

## БЛОК 3: ПРОБЛЕМЫ КЛИЕНТА (известные боли)

### Экспертиза: SEO-анализ + Психология продаж + Копирайтинг

**Задача:** Показать, что мы знаем их ситуацию лучше, чем они сами.

### Исследование: Что ищут владельцы (из SEO-ядра)

По данным поисковых запросов, владельцы сайтов приходят с этими проблемами:
1. "медленный сайт что делать" (~90 запр/мес) → **Скорость**
2. "сайт не работает на телефоне" (~70) → **Мобильная версия**
3. "обновить дизайн сайта" (~50) → **Устаревший вид**
4. "улучшить конверсию сайта" (~80) → **Мало заявок**
5. "переделать старый сайт" (~60) → **Общее устаревание**
6. "русификация сайта закон 2026" (~120) → **Законодательство**

### Контент блока

**Заголовок секции:**  
`Знакомо?`

**Подзаголовок:**  
`Проблемы, с которыми к нам приходят чаще всего`

**Формат:** 4-6 карточек с иконками, каждая описывает боль

| # | Боль | Заголовок карточки | Описание |
|---|------|--------------------|----------|
| 1 | Скорость | «Сайт грузится вечность» | Клиенты уходят, не дождавшись загрузки. 53% посетителей уходят, если сайт грузится дольше 3 секунд. Каждая секунда промедления — минус 7% конверсии. |
| 2 | Мобильная версия | «На телефоне всё ломается» | 70% трафика — мобильные устройства. Если сайт не адаптирован, вы теряете 7 из 10 потенциальных клиентов. Яндекс понижает такие сайты в выдаче. |
| 3 | Дизайн | «Сайт выглядит как из 2018» | Устаревший дизайн снижает доверие. 75% пользователей судят о надёжности компании по внешнему виду сайта. Конкуренты с современным дизайном забирают ваших клиентов. |
| 4 | Заявки | «Трафик есть — заявок нет» | Посетители приходят, но ничего не делают. Проблема не в рекламе — в сайте. Плохой UX, непонятная навигация, отсутствие CTA — и заявки утекают к конкурентам. |

**Дизайн:** Тёмные карточки с оранжевыми акцентами, каждая с анимацией появления при скролле. Иконки — стилизованные под LEGO-кубики (связь с общей темой).

## БЛОК 4: СКРЫТЫЕ ПРОБЛЕМЫ (о которых не знают)

### Экспертиза: Технический SEO + Веб-разработка + Аналитика

**Задача:** Создать "aha-момент" — показать экспертизу через знание скрытых проблем. Углубить ощущение "мне точно нужна помощь".

### Исследование: Реальные скрытые проблемы

На основе технического аудита типичных сайтов МСБ:

| # | Скрытая проблема | Почему скрытая | Финансовый ущерб |
|---|------------------|----------------|-----------------|
| 1 | **Core Web Vitals ниже нормы** | Google/Яндекс понижают в выдаче, владелец не видит связи. LCP >2.5 сек, INP >200мс, CLS >0.1 | Потеря 20-40% органического трафика |
| 2 | **Не настроена аналитика** | CRM не связана с Метрикой, нет целей, невозможно понять, откуда лиды и какой ROMI | Сливание рекламного бюджета вслепую |
| 3 | **Битые ссылки и ошибки 404** | Поисковые роботы тратят краулинговый бюджет на битые страницы вместо индексации нового контента | Потеря до 40% трафика (данные аудитов) |
| 4 | **Нет структурированных данных** | Сайт не получает расширенные сниппеты в выдаче (звёздочки, цены, FAQ), CTR падает | Конкуренты с rich snippets получают в 2-3 раза больше кликов |
| 5 | **Устаревшие протоколы безопасности** | HTTP вместо HTTPS, или сертификат просрочен. Браузеры показывают "Небезопасно" | Мгновенная потеря доверия, штрафы Google |
| 6 | **JavaScript-блокирующий рендеринг** | Сторонние скрипты (счётчики, виджеты, чат-боты) грузятся синхронно и тормозят отрисовку | Увеличение LCP на 2-5 сек |

## БЛОК 5: КАК МЫ РЕШАЕМ (+ 3D-анимация с кубиками)

### Экспертиза: UX + Анимация/3D + Процесс + Копирайтинг

**Задача:** Показать, что у нас есть системный подход. Метафора: из хаоса → в порядок (кубики складываются в стройную башню).

### 3D-анимация: техническое решение

**Выбранный подход: Video Scrubbing (scroll-synced video)**

Причины выбора:
- Фотореалистичный 3D-эффект
- Контроль прокрутки: видео играет вперёд при скролле вниз, реверс при скролле вверх
- Работает на мобильных
- Не требует runtime-библиотек для 3D

**Как создать видео:**

Вариант А — Blender (рекомендуемый для максимального качества):
1. Создать 3D-модель: механическая клешня + цветные кубики (LEGO-стиль)
2. Анимация: клешня берёт кубики и аккуратно ставит один на другой, выстраивая ровную башню
3. Рендер: 120-180 кадров, разрешение 1080p, прозрачный/тёмный фон
4. Экспорт: MP4 (H.264) или последовательность JPEG

Вариант Б — AI-генерация видео (быстрее, но менее контролируемо):

Промпт для Runway Gen-3 / Kling:
```
Overhead cinematic shot of a robotic mechanical claw with metallic silver finish,
precisely stacking colorful LEGO-style cubes (orange, white, dark gray) into a
tall perfect tower on a dark surface. Smooth precise robotic movements,
industrial lighting with soft shadows, 4K ultra-realistic 3D style, slow motion.
Camera slowly orbiting around the tower as it grows.
```

Промпт для Pika / Sora:
```
Close-up of a mechanical arm carefully placing orange and white building blocks
one by one into a perfectly aligned tower. Minimalist dark background. The blocks
have a matte LEGO-like texture. Smooth, satisfying, ASMR-style movement.
Studio lighting. Photorealistic 3D render quality.
```

**Реализация scroll-синхронизации:**

Современный подход (WebCodecs API для плавности):
```javascript
// Видео привязано к позиции скролла секции
const section = document.querySelector('#solution');
const video = document.querySelector('#solution-video');

function onScroll() {
  const rect = section.getBoundingClientRect();
  const sectionHeight = section.offsetHeight - window.innerHeight;
  const scrolled = -rect.top;
  const progress = Math.max(0, Math.min(1, scrolled / sectionHeight));
  video.currentTime = progress * video.duration;
}

window.addEventListener('scroll', () => requestAnimationFrame(onScroll));
```

Альтернатива: JPEG-sequence (для максимальной плавности на мобильных):
- Экспорт 120 кадров из Blender как PNG
- Предзагрузка в массив Image()
- Отрисовка текущего кадра на Canvas по scroll position

| Шаг | Кубик | Название | Описание |
|-----|-------|----------|----------|
| 1 | Оранжевый | **Аудит** | Находим все проблемы: скорость, UX, SEO, конверсия. Вы видите отчёт с цифрами. |
| 2 | Белый | **Прототип** | За 3 дня показываем AI-прототип — вы видите результат до оплаты. |

**Визуальная механика:**
- По мере скролла через секцию кубики складываются в башню (видео играет)

- При скролле назад — кубики разбираются, карточки исчезают

## БЛОК 6: ТАРИФЫ И ЦЕНЫ

### Экспертиза: Ценообразование + Психология продаж + Конверсия

### Исследование: оптимальная структура

**Принципы ценовой психологии:**
1. **Якорь:** Сначала показать, сколько стоит новый сайт (300-500к), потом нашу цену
2. **Decoy:** Средний тариф — целевой, верхний — якорь
3. **Loss aversion:** Показать, сколько денег клиент ТЕРЯЕТ каждый месяц с текущим сайтом
4. **Charm pricing:** Базовый и средний — charm (99к, 199к), Premium — круглое (300к)

### Для 3D-анимации: пошаговый план

**Если Blender:**
1. Установить Blender (бесплатно)
2. Смоделировать кубики (простая геометрия) + механическую клешню
3. Анимировать: клешня берёт кубик → ставит на башню (6 итераций)
4. Рендер: 120-180 кадров, 1080p, тёмный фон
5. Экспорт: JPEG-sequence или MP4
6. Интеграция в сайт через Canvas + scroll listener

**Если AI-видео (быстрее):**
1. Создать видео через Runway Gen-3 / Kling (промпты выше)
2. Обрезать до нужного фрагмента (10-15 сек)
3. Конвертировать в WebM (лучшее сжатие)
4. Реализовать scroll-synced playback

**Если CSS/JS (самое быстрое, но менее эффектно):**
1. Создать CSS 3D-кубики с perspective
2. Анимировать transform: translateY при скролле
3. SVG-клешня с CSS анимацией
4. Полный контроль, но менее реалистично

### Рекомендация по приоритету создания 3D:
Начать с CSS/JS варианта (быстро, работает), параллельно создать Blender-видео для замены позже.

## КОПИРАЙТИНГ: ОБЩИЕ ПРИНЦИПЫ

### Тон коммуникации
- **Уверенный, но не агрессивный.** "Мы знаем как" вместо "Вы делаете всё неправильно"
- **Конкретный.** Цифры > обещания. "73% клиентов уходят" > "Многие клиенты уходят"
- **Эмпатичный.** Понимаем боль, не давим на неё
- **Простой.** Без жаргона. "Сайт грузится быстрее" > "Оптимизация LCP метрики"

### Формула каждого блока:
1. Проблема/вопрос → 2. Усиление (цифры/факты) → 3. Решение/следующий шаг

### Избегать:
- "Мы — команда профессионалов" (клише)
- "Индивидуальный подход" (пусто)
- "Комплексные решения" (непонятно)
- "Широкий спектр услуг" (размывает фокус)
